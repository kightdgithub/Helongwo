Hexo插件之Hexo-deployer-git

Hexo插件之Hexo-deployer-git

安装好插件之后来看看Hexo项目目录下的 “.gitignore” 文件;
.deploy*/: 表明 .deploy_git 目录下的文件是忽略的,Hexo会将每次生成的静态存放与.deploy_git目录下再提交;
此插件也是会自动的做 GIT ADD,COMMIT的操作,可以从Hexo项目目录下的 node_modules –> hexo-deployer-git –> lib下的deployer.js中大致看出;

当我们为Gitc仓库单独配置User.Name和User.Email时(见Git多账号配置),可能在 hexo -d 时会提示要设置Name和Email,估计是插件默认会使用全局的Name和Email而不是单独仓库的,此时我们需要进入到.deploy_git 目录下手动在设置一次,这样插件检测到已有用户名和邮件地址,并不会在提醒;

需要注意的是,默认的仓库配置在Hexo项目目录下的_config.yml中可以配置;
