Linux版本之Unity3d开发环境配置

Linux版本之Unity3d开发环境配置

最近才了解到unity3d从5.0版本左右以来一直有为linux发布测试版本,下载地址为:https://forum.unity3d.com/threads/unity-on-linux-release-notes-and-known-issues.350256
临近放假回家,身旁暂无笔记本,想起家里有台老旧PC,索性弄个portable linux回家耍算了.
硬盘不动(数据无价不解释),身边只有一个闲置USB2.0和一个class10的MicroSD卡,大小都为16G,条件有点限制(Unity场景就不要想了,vscode写点代码什么的还是可以的).

网上搜了一下,大概流程不复杂,工具也不算多,这里选用工具版本有:

1,Ubuntu16.04 LST(大小1.6G),(话说ubuntu18都已经放弃unity桌面了),这里选用16.04长期支持版本,考虑到稳定性和对旧电脑低配友好,原生的wifi以及网卡适配器也是可以直接识别我的网卡,无需额外驱动,fcitx有较好用的google拼音输入法了,并且unity官方也建议ubuntu;

2,DotNet Core 2.2(约270M), (可以查看ubuntu16.04官方安装教程)[https://dotnet.microsoft.com/download/linux-package-manager/ubuntu16-04/sdk-2.2.103];

3,mono 4.2.1,(查看官方安装教程)[https://www.mono-project.com/download/stable/#download-lin];

4,unity-editor-5.6.0xf3Linux(deb为2.5G,sh自解压为2.4G),也有2017和2018版本,体积略大, (有deb包和sh自解压包)[https://forum.unity.com/threads/unity-on-linux-release-notes-and-known-issues.350256/];

5,vscode 1.3.2,官网(下载)[https://code.visualstudio.com/]安装;

具体步骤

1,Ubuntu的安装
SD卡做系统盘,usb2.0作为存储盘; 在window10的环境下使用软碟通(ultraiso)[https://cn.ultraiso.net/chanpinjieshao.html]来制作USB硬盘启动盘(也叫liveCD),(具体步骤看百度)[https://www.baidu.com/s?ie=utf-8&f=8&rsv_bp=1&rsv_idx=1&tn=baidu&wd=%E8%BD%AF%E7%A2%9F%E9%80%9A%20usb%20ubunto%20liveCD&oq=%25E8%25BD%25AF%25E7%25A2%259F%25E9%2580%259A&rsv_pq=c905ff6b00007f1e&rsv_t=c2f13Dnl6BzQ8XDFe0FT6WHGudyZzrwqYd7ZNFy%2FZY4IbU5BNL0XrsjjgIE&rqlang=cn&rsv_enter=1&inputT=11611&rsv_sug3=93&rsv_sug1=61&rsv_sug7=100&rsv_sug2=0&rsv_sug4=13032]这里不提.

将SD卡加读卡器,U盘插入电脑,弄清楚他们在bios中启动选项的名称,进入U盘的Ubuntu liveCD模式(数据无价,如果确定不了,请花点时间确认你的bios能否禁用硬盘的stat接口,否则拆硬盘吧>_>);

拔掉你的网线或者断掉网络连接,因为默认Ubuntu安装是联网的,并且使用默认源来更新(虽然也会按照你的时区选择跟换服务器,不过有aliyun源在,你说呢>_>);

之后自己选择自定义安装到的分区(16G大小就不考录按照目录分了 >_<!!),一般看大小和名字可以区别，swap正常来讲Ubuntu16有2G以上内存不会频繁使用swap的,家里旧的电脑还有4G呢,留个512;

之后就是15~20分钟的等待,毕竟现在系统盘是个class10的SD卡(-_-!);

完成后Ubuntu会提醒可以重启进入新系统了,并且还会告诉你先拔出liveCD盘(至少在我看来,这话不是废话,因为第一次没有拔,结果貌似在bootload的时候直接重启,第二次拔掉正常,不知道是不是个例),这个步骤你可能会遇到grub secure情况,(请百度)[https://www.baidu.com/s?ie=utf-8&f=8&rsv_bp=1&rsv_idx=1&tn=baidu&wd=ubuntu16.04%20grub%20rescue&oq=ubuntu16.04%2520grub%2520rescu&rsv_pq=9ceec7ca00011ed3&rsv_t=ad81dVfGg5x%2BF28KzjcYbUQFnv4iqlAUayBqeQ11QO46TEtfvRiKkUAERtY&rqlang=cn&rsv_enter=1&inputT=193&rsv_sug3=148&rsv_sug2=0&rsv_sug4=520];

在你进入真正的系统盘后,就可以Ubuntu自带图形工具Disk格式化掉你的U盘来作为新存储盘了(需要格式化为EXT格式),因为Ubuntu16.04大概用掉你SD的一半空间,而unity安装包解压后约7.9G,放在一起的话,恩….，并且之后如果使用deb来安装还不一定成功(估计就是容量问题,图标都生成,提示你剩余容量700M之后什么都打不开),另外一种方式就是使用dpkg的 “–instdir” 指定目录安装到U盘,各种搔报错,折腾不起.

你还可能在格式化U盘的时候碰到这个问题: “Warning: The driver descriptor says the physical block size is 2048 bytes, but Linux says it is 512 bytes.”,(用dd擦除解决,看回答3)[https://www.helplib.com/ubuntu/article_156188]

2,跟换阿里云的镜像源,修改源配置文件sudo gedit /etc/apt/sources.list之前别忘记备份,之后你可以选择使用aptitud来作为包管理工具,sudo aptitude update; sudo aptitude upgrade来进行跟新,sudo aptitude clean可以清楚缓存包,对于只有16G的系统盘当然不能浪费在这上面;

3,在安装了vim,fcitx等等软件之后,在你认为能基本舒服使用的时候,可以开始正式的环境搭建了.

基础的DotNet Core是不可少的,进入(官网)[https://dotnet.microsoft.com/download/linux-package-manager/ubuntu16-04/sdk-2.2.103]可以看到只有五条命令;

运行

4,安装mono 4.2.1,(查看官方安装教程)[https://www.mono-project.com/download/stable/#download-lin],步骤也很简单;

这个网上说可以在vscode中按F12看到源码,但是并没有什么卵用(>_>无反应)

5,官网(下载)vscode[https://code.visualstudio.com/]安装;

可以考虑C#,unity-debug,unity-tool等插件.

6,unity-editor-5.6.0xf3Linux(sh自解压包)[https://forum.unity.com/threads/unity-on-linux-release-notes-and-known-issues.350256/];
这个很方便,官方称为”Platform-Agnostic Self-Extracting Shell Script”,不只是给ubuntu用的,但是ubuntu也可以解压使用(解压位置全英文且分区为ext文件格式);
deb包也是一样,不过对于提示确实包依赖,不建议使用 “-f” 参数的粗暴方式,使用

来察看缺少的依赖，手动安装后在装unity的deb包,否则apt工具可能在安装unity3d时提示包缺失时的同时还丢失安装信息,导致apt-get purge 失败,无法移除原有包;

安装完成后,启动文件在/opt/Unity/Editor/目录下的Unity,解压办为解压位置的Editor目录下Unity,直接启动的话,一般还提示你 “Chromium Embedded Framework requires that the file ‘/opt/Unity/Editor/chrome-sandbox’ is owned by root and hase permission 4755”,修改同目录下的chrome-sandbox权限即可,可以参考(这篇文章:https://www.plop.at/en/ploplinux/desktop/unity5.html)[https://www.plop.at/en/ploplinux/desktop/unity5.html];

之后就可以正常启动了,在unity prefrence设置C#编辑器为vscode既可以编写代码了.

Finally ~~~ U盘和SD卡都分别还剩约7个G,应该足够耍了.
